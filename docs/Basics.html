Basics













Lol Admin is a simple to use admin tool for your game. It is designed to be a simple to use and easy to understand. and its a good tool for moderation and maybe a little fun!
Lol Admin is automatically setup when you insert it into your game but we reccomend customize it to your needs.





Customization








To customize the admin tool, you will need to create a config file. Luckily it already comes with one! So once you have it in your game, open up the model and you should see a script called: Setup. If you dont see it, try removing the model and re-adding it. If you do see it, you can open it up and edit the script called Configuration. This script is where you will set up the admin tool. Now the script should look something like this:







local config = {}

config.Configuration = {

--Game HeadAdmin's
HeadAdmin = {

},

--Game HeadAdmin's
Admin = {

},

--Game HeadAdmin's
Mod = {

},

--Game HeadAdmin's
VIP = {

},

--Role for vip server owners
VIPServerOwnersRole = "Mod",

--Role for the game owner friends
OwnerServerFriendsRole = "NonAdmin",

--Command prefix
Prefix = "!"

}

return config


















































Now if we want to add lets say a new Admin. We will need to add them to the HeadAdmin table. To do this, we will need to open up the Configuration script and add them to the HeadAdmin table. Lets add a new string with a player name like this "RacerDreng". It should look something like this at last







local config = {}

config.Configuration = {

--Game HeadAdmin's
HeadAdmin = {
"RacerDreng"
},

--Game HeadAdmin's
Admin = {

},

--Game HeadAdmin's
Mod = {

},

--Game HeadAdmin's
VIP = {

},

--Role for vip server owners
VIPServerOwnersRole = "Mod",

--Role for the game owner friends
OwnerServerFriendsRole = "NonAdmin",

--Command prefix
Prefix = "!"

}

return config















































Now once you have added the new HeadAdmin, you will need to restart the server. And of course this will work with all other Role tabs as well.
Now that you understand how to setup the modules configuration you can move on to the next step!
